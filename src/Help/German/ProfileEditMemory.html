Profileditor-Dialogseite "Arbeitsspeicher"

Auf dieser Seite kann eingestellt werden, wie viel und welche Typen von Arbeitsspeicher DOSBox dem Spiel zur Verfügung stellen soll.

  • Arbeitsspeicher (in MB)
    In dieser Eingabezeile können Sie die Gesamtmenge an Arbeitsspeicher (gemessen in MB) angeben, die DOSBox dem Spiel zur Verfügung stellen soll.
  • XMS-Speicher aktivieren
    Wenn Sie diese Option aktivieren, wird DOSBox dem in Anwendungen XMS-Speicher zur Verfügung stellen. Wenn das Spiel, welches Sie ausführen wollen XMS-Speicher unterstützt, müssen Sie lediglich diese Option aktivieren. (Die Installation und Konfiguration eines Speichermanagers wie EMM386 in DOSBox ist nicht erforderlich.) Da diese Option Spiele, die XMS-Speicher nicht unterstützen, nicht stört, können Sie sie immer aktiviert lassen (es sei denn ein Spiel hat explizit Probleme mit der XMS-Emulation von DOSBox).
  • EMS-Speicher aktivieren
    Wenn Sie diese Option aktivieren, wird DOSBox dem in Anwendungen EMS-Speicher zur Verfügung stellen. Wenn das Spiel, welches Sie ausführen wollen EMS-Speicher unterstützt, müssen Sie lediglich diese Option aktivieren. (Die Installation und Konfiguration eines Speichermanagers wie EMM386 in DOSBox ist nicht erforderlich.) Da diese Option Spiele, die EMS-Speicher nicht unterstützen, nicht stört, können Sie sie immer aktiviert lassen (es sei denn ein Spiel hat explizit Probleme mit der EMS-Emulation von DOSBox).
  • UMB-Speicher aktivieren
    Wenn Sie diese Option aktivieren, wird DOSBox dem in Anwendungen UMB-Speicher zur Verfügung stellen. Wenn das Spiel, welches Sie ausführen wollen UMB-Speicher unterstützt, müssen Sie lediglich diese Option aktivieren. (Die Installation und Konfiguration eines Speichermanagers wie EMM386 in DOSBox ist nicht erforderlich.) Da diese Option Spiele, die UMB-Speicher nicht unterstützen, nicht stört, können Sie sie immer aktiviert lassen (es sei denn ein Spiel hat explizit Probleme mit der UMB-Emulation von DOSBox).
  • Loadfix verwenden
    Wenn ein Spiel nicht korrekt startet und die Symptome vermuten lassen, dass das Spiel Probleme mit der Speicherverwaltung hat, so können Sie testweise diese Option verwenden. Loadfix verringert die Menge an verfügbarem konventionellem Arbeitsspicher und kann dadurch Probleme beheben, die dadurch entstehen, dass mehr konventioneller Arbeitsspeicher verfügbar ist, als das Spiel verwalten kann.
  • DOS32A Extender verwenden (*)
    Wenn das zu startende Spiel einen DOS Extender wie z.B. DOS4/GW verwenden, können Sie diesen durch das Aktivieren dieser Option gegen den moderneren und schnelleren DOS Extender DOS32A ersetzen. D-Fend Reloaded zeigt durch ein Symbol rechts neben dieser Option an, ob das Spiel einen DOS Extender verwendet oder nicht. Sie sollten diese Option nur aktivieren, wenn das Spiel auch einen DOS Extender verwendet; andernfalls kann es passieren, dass das Spiel nicht mehr korrekt startet.

Für die Verwendung der mit (*) markierten Option wird der DOS Extender DOS32A aus dem FreeDOS-Paket verwendet. Das heißt, dass diese Option nur dann zur Verfügung steht, wenn FreeDOS auf dem System verfügbar ist. Sie können den Pfad zu Ihrer FreeDOS-Installation auf der Dialogseite FreeDOS des Programmeinstellungen-Dialogs angeben. Der Informationstext unter der Option wird in grau dargestellt, wenn FreeDOS verfügbar ist und er wird in rot dargestellt, wenn die Option nicht verwendet werden kann, da kein FreeDOS vorhanden ist.
