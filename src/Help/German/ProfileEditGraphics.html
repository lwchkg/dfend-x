Profileditor-Dialogseite "Grafikkarte"

Auf dieser Seite können Sie die virtuelle Grafikkarte, die DOSBox bzw. ScummVM dem Spiel zur Verfügung stellt, konfigurieren.

Für DOSBox-Profile

  • Fenster-Auflösung
    In diesem Auswahlfeld können Sie die Auflösung wählen, die DOSBox verwenden soll, wenn das Spiel als Fenster auf dem Windows Desktop läuft. "Original" bedeutet dabei, dass DOSBox ohne jegliche Skalierung die Auflösung verwenden soll, die das Spiel vorgibt. In allen anderen Fällen skaliert DOSBox die Ausgabe entsprechend. (Siehe dazu auch die Option "Skalierung" weiter unten.)
  • Vollbild-Auflösung
    In diesem Auswahlfeld können Sie die Auflösung wählen, die DOSBox verwenden soll, wenn das Spiel im Vollbildmodus läuft. "Original" bedeutet dabei, dass DOSBox ohne jegliche Skalierung die Auflösung verwenden soll, die das Spiel vorgibt. In allen anderen Fällen skaliert DOSBox die Ausgabe entsprechend. (Siehe dazu auch die Option "Skalierung" weiter unten.) Zur Darstellung der gewünschten Auflösung im Vollbildmodus wechselt DOSBox ggf. temporär die Windows-Auflösung.
  • Vollbild beim Start
    Wenn diese Option aktiviert ist, startet DOSBox im Vollbildmodus; andernfalls startet es als normales Fenster auf dem Windows Desktop.
  • Doublebuffering verwenden
    Durch das Aktivieren dieser Option können Sie DOSBox anweisen, im Vollbildmodus Doublebuffering zu verwenden. Die Verwendung von Doublebuffering kann diverse Grafikprobleme lösen aber auch neue verursachen, in Abhängigkeit von Ihrem Windows-Grafiktreiber, der verwendeten Betriebssystemversion und dem Spiel, das Sie ausführen möchten.
  • Seitenverhältnis korrigieren
    Die Verwendung dieser Option ist sinnvoll, wenn Sie ein Spiel mit der Auflösung von 320x200 Pixeln auf einem modernen Bildschirm darstellen woll. Bei einer Verwendung von 320x200 Pixeln wurden früher auf Röhrenmonitoren nicht quadratische Pixel verwendet; auf heutigen LCD-Monitoren sind die Pixel aber bauartbedingt stets quadratisch, so dass ein 320x200 Pixel Bild verzerrt wirkt. Die Option zu Korrektur des Seitenverhältnisses streckt in dem oben beschriebenen Fall das Bild von 320x200 auf 320x240 Pixel. In Abhängigkeit von dem verwendeten Interpolationsalgorithmus sieht das Bild dann meistens wesentlich besser aus.
  • Glide-Emulation (*)
    Wenn diese Option aktiviert ist, stellt DOSBox dem Spiel eine Glide-Emulation zur Verfügung.
  • Darstellungsmethode
    In dieser Aufwahlbox können Sie festlegen, wie DOSBox den Bildschirminhalt der virtuellen Grafikkarte an die reale Grafikkarte übertragen soll. Die Einstellung "surface" ist zwar verhältnismäßig langsam, sollte aber mit praktisch allen Windows-Grafiktreibern funktionieren. Sollten bei einem Spiel Grafikprobleme auftreten (inklusive der Tatsache dass Ein- und Ausblendeeffekte extrem langsam ablaufen), können Sie hier andere Einstellungen ausprobieren; so sollte z.B. "Overlay" mittlerweile von praktisch allen Grafikkarten unterstützt werden und ist wesentlich schneller.
  • Grafikkarte
    In dieser Auswahlbox können Sie wählen, welchen Grafikkartentyp DOSBox emulieren soll. Bei den meisten Spielen sollte die Verwendung von "vga" funktionieren (da VGA-Karten ebenfalls die niedrigeren Grafikmodi der früheren CGA- und EGA-Karten anbieten). Sollten Sie jedoch ein Spiel ausführen wollen, welches explizit z.B. nur mit CGA-Karten zusammenarbeitet, so können Sie hier einen anderen Grafikkartentyp festlegen.
  • Pixel-Shader (*)
    Über diese Asuwahlbox können Sie das zu verwendende Pixel-Shader Programm für DOSBox festlegen. Die hier angezeigten Programme sind fx-Dateien im Shaders-Unterverzeichnis des DOSBox-Verzeichnisses der gewählten DOSBox-Installation.
  • Einstellungen für Grafikkartentyp "VGA" (*)
    In dieser Box können Sie den genauen Typ der emulierten VGA-Karte einstellen (d.h. diese Einstellung kommen nur dann zum Tragen, wenn Sie als Grafikkartentyp "vga" gewählt haben). Diese Einstellungen können bei DOS-Spielen, die einen bestimmten VGA-Chipsatz voraussetzen, hilfreich sein.
  • Skalierung
    In diesm Aufwahlfeld können Sie den Algorithmus bestimmen, nach dem Bildinhalte, die in einer höheren Auflösung auf dem physikalischen Bildschirm dargestellt werden sollen, als es das Spiel vorgibt, hochskaliert werden sollen. Die jeweils beste Skalierungsmethode hängt von dem Spiel, das Sie ausführen wollen, ab. Allgemein gilt, dass Algorithmen, die weiter oben in der Liste aufgeführt sind, eher schneller sind (d.h. weniger Rechenleistung benötigen) und Algorithmen, die weiter unten stehen meist eher bessere Ergebnisse, d.h. glattere Bilder, liefern.
  • Frameskip
    In diesem Eingabefeld können Sie festlegen, wie viele Bilder, die das Spiel darstellen möchte, DOSBox jeweils zwischen der realen Darstellung von zwei Bildern auf dem Bildschirm auslassen soll. Ein höherer Frameskipwert bedeutet, dass weniger Daten an die reale Grafikkarte übertragen werden müssen und damit die gesamte Darstellung schneller ablaufen kann, d.h. dass das Spiel dadurch schneller wird. Die Bedeutung des Frameskips ist nicht an die Anzahl der Bilder, die das Spiel pro Sekunde berechnet (fps, frames per second) gekoppelt, sondern hängt mit der virtuellen Darstellungsrate des virtuellen Monitors in DOSBox zusammen. Der virtuelle Monitor arbeitet in den meisten Auflösungsstufen mit 70fps und im 640x480 VGA-Modus mit 60fps. Ein Frameskip von 1 bedeutet damit, dass 35 bzw. 30 Bilder pro Sekunde an die reale Grafikkarte übertragen werden, ein Frameskip von 2 bedeutet damit, dass 23.3 bzw. 20 Bilder pro Sekunde an die reale Grafikkarte übertragen werden und schließtlich werden bei einem Frameskip von 3 je nach Auflösungseinstellung 17.5 bzw. 15 Bilder pro Sekunde übertragen.
  • Zeilen im Textmodus (*)
    In dieser Box können Sie festlegen, wie viele Textzeilen DOSBox im normalen Textmodus (z.B. während die Eingabeaufforderung angezeigt wird) verwenden soll. Der Vorgabewert für diese Option ist 25.

Die mit (*) markierten Optionen sind nur sichtbar, wenn Sie das zugehörige DOSBox CVS-Feature auf der Dialogseite DOSBox CVS-Features des Programmeinstellungen-Dialogs aktiviert haben. Die offizelle DOSBox 0.72 Version, die mit D-Fend Reloaded ausgeliefert wird, unterstützt diese zusätzlichen Optionen nicht.

Für ScummVM-Profile

  • Filter
    In diesem Feld können Sie auswählen, nach welcher Methode ScummVM die Spielegrafiken auf die physikalische Auflösung des Systems hochskalieren soll.
  • Darstellungsart
    In diesem Feld können Sie die zu verwendende Grafikkartenemulation festlegen. Im Normalfall sollte der Vorgabewert "default" beibehalten werden.
  • Vollbild beim Start
    Wenn diese Option aktiviert ist, startet ScummVM im Vollbildmodus; andernfalls startet es als normales Fenster auf dem Windows Desktop.
