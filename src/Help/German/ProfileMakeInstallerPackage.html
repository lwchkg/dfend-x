Installer-Paket erstellen

D-Fend Reloaded kann exe-Installationspakete von bestehenden Spielen erstellen. Diese Pakete enthalten alle Spiele-Dateien (Programmdateien, D-Fend Reloaded Profil, Datenordner, aufgezeichnete Screenshots usw.) und können archiviert und später wieder installiert werden.

Um exe-Installationspakete zu erstellen, benutzt D-Fend Reloaded das Opensource Installationsprogramm-Erstellungssystem Nullsoft scriptable install system (NSIS). D-Fend Reloaded erstellt nsi Skripte, aus denen dann NSIS eine exe-Datei erstellt. Das heißt, Sie müssen NSIS auf Ihrem System installiert haben, um diese Funktion nutzen zu können. (Sie müssen den NSIS-Installationspfad nicht explizit in D-Fend Reloaded angeben, da er aus der Registry ausgelesen werden kann.)

Paket-Einstellungen

In dieser Box können Sie auswählen, ob D-Fend Reloaded nur das nsi Skript zu späteren manuellen Erstellung des Installers erstellen soll oder aber das nsi Skript erstellen soll und dann unmittelbar mit Hilfe von NSIS einen Installer daraus erstellen soll. (In den meisten Fällen dürfte die zweite Option ihr Wunsch sein.)

Dateiname des Paketes

In diesem Eingabefeld können Sie den Namen des zu erstellenden exe-Installers angeben. Die Datei muss sich in dem Basisordner (siehe Dialogseite Verzeichnisse im Programmeinstellungen-Dialog) befinden, da in dem nsi-Skript relative Pfade verwendet werden, die nur dann von NSIS richtig interpretiert werden können, wenn sich das Skript in diesem Ordner befindet. (Nach der Erstellung kann der exe-Installer in ein beliebiges Verzeichnis kopiert werden und kann auch von einem beliebigen Verzeichnis aus gestartet werden.)
