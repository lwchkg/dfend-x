Programmeinstellungen-Dialogseite "Wine-Unterstützung"

Diese Dialogseite ist nur sichtbar, wenn Sie den Konfigurationsmodus "Vollständig" unten im Programmeinstellungen-Dialog gewählt haben.

Sowohl D-Fend Reloaded als auch die Windows-Version von DOSBox können in einer Wine-Umgebung unter Linux ausgeführt werden. Von daher besteht eigentlich kein Bedarf, spezielle Einstellungen zur Verwendung von D-Fend Reloaded unter Wine vorzunehmen. D-Fend Reloaded kann jedoch, wenn unter Wine eingesetzt, auch direkt mit der Linux-Version von DOSBox zusammenarbeiten und bietet auch sonst einige Komfort-Funktionen (wie die Erstellung von Shellskripten) an, die beim Einsatz unter Linux nützlich sein können.

  • Wine-Modus aktivieren
    Durch das Aktivieren bzw. deaktivieren dieser Auswahlbox, lässt sich die gesamte Wine-Unterstützung in D-Fend Reloaded ein- bzw. ausschalten. Wenn die Wine-Unterstützung durch diese Auswahlbox deaktiviert ist, werden alle weiteren Einstellmöglichkeiten auf dieser Dialogseite durch D-Fend Reloaded im Betrieb ignoriert.
    Sie können die selbe D-Fend Reloaded Installation unter Windows und unter Wine verwenden, ohne dafür die Konfiguration jedes Mal ändern zu müssen): Wenn Sie D-Fend Reloaded mit einem der Kommandozeilenparametern "WineMode" oder "WineSupportEnabled" aufrufen, können Sie damit die Einstellung des Schalters "Wine-Modus aktivieren" auf "ein" setzen. Wenn Sie D-Fend Reloaded mit einem der Kommandozeilenparametern "WindowsMode", "NoWineSupport" oder "WineSupportDisabled" aufrufen, können Sie damit die Einstellung des Schalters "Wine-Modus aktivieren" auf "aus" setzen. Dabei gelten die durch den Parameter vorgegebenen Einstellungen nur so lange D-Fend Reloaded läuft. Wenn Sie es erneut ohne Parameter aufrufen, gilt wieder der ursprüngliche Wert dieser Einstellung, d.h. durch den Aufruf mit einem der oben gegannten Parameter wird die Konfiguration nicht dauerhaft verändert. Bei der Auswertung der Parameter wird die Größ- und Kleinschreibung nicht beachtet. Außerdem kann den Parametern nach eigenem Geschmack auch ein "-" oder ein "/" vorangestellt werden (d.h. auch "-winemode" würde korrekt erkannt werden).
    (Vorgabewert für diese Option: aus)
  • Wine-Laufwerke auf Linux-Pfade rückabbilden (remappen)
    In dieser Liste können Sie angeben, auf welche Linux Pfadnamen (/mnt/sta3, /mnt/stb1 usw.) die Wine-Laufwerksbuchstaben (C: D: usw.) beim Erstellen der Conf-Dateien rückabgebildet werden sollen.
  • Mount-Befehle remappen
    Ist diese Option aktiviert, so werden die Pfade in den Mount-Befehlen auf die in der Remapping-Liste angegebenen Linux-Pfade rückabgebildet.
    (Vorgabewert für diese Option: aus)
  • Screenshot-Ordner remappen
    Ist diese Option aktiviert, so wird der Pfad für den Screenshots-Ordner gemäßg der Remapping-Liste auf den zugehörigen Linux-Pfade rückabgebildet.
    (Vorgabewert für diese Option: aus)
  • Keymapping-Datei remappen
    Ist diese Option aktiviert, so wird der Pfad zu der Keymapping-Datei gemäßg der Remapping-Liste auf den zugehörigen Linux-Pfade rückabgebildet.
    (Vorgabewert für diese Option: aus)
  • DOSBox-Pfad remappen
    Ist diese Option aktiviert, so wird der Pfad zur DOSBox-Installation selbst gemäßg der Remapping-Liste auf den zugehörigen Linux-Pfade rückabgebildet.
    (Vorgabewert für diese Option: aus)
  • Menüpunkt zur Erstellung von Verknüpfungen erstellt Linux Shellskripte
    Wenn diese Option aktiviert ist, so können über den Menüpunkt "Verknüpfung erstellen" im "Profil"-Menü Linux Shellskripte zum direkten Starten von DOSBox mit dem jeweiligen Profil erstellt werden. Andernfalls erstellt die "Verknüpfung erstellen"-Funktion Windows-Verknüpfungen.
    (Vorgabewert für diese Option: aus)
  • Shellskript-Preambel
    Wenn die Funktion zur Erstellung von Linux Shellskripten statt Windows-Verknüpfungen aktiviert ist, so wird beim Erstellen eines solchen Skriptes die in dieser Eingabezeile angegebene Präambel in das Skript eingefügt.
    (Vorgabewert für diese Option: "#!/bin/bash")
