Nach Updates suchen

Durch das Anklicken dieses Menüpunkts starten Sie eine manuelle Suche nach Updates für D-Fend Reloaded. Sie können auf der Nach Updates suchen Dialogseite des of the Programmeinstellungen-Dialogs auch einstellen, dass D-Fend Reloaded in regelmäßigen Abständen automatisch nach Updates sucht.

Die Updateüberprüfung überträgt keine persönlichen Daten ins Internet. Bei der Updateüberprüfung wird lediglich der Inhalt einer auf der D-Fend Reloaded Homepage angebotenen Textdatei heruntergeladen und ausgewertet. Wenn die Versionsnummer, die in dieser Textdatei angegeben ist, größer als die Versionsnummer der aktuellen Installation ist, bietet die Updateprüfung an, den in der zweiten Zeile der Textdatei angegebenen Updateinstaller herunter zu laden und auszuführen.

Dateils zur Funktionsweise der Update-Überprüfung

Der Update-Checker läd folgende Datei von der D-Fend Reloaded Hompage auf Sourceforge herunter: "http://dfendreloaded.sourceforge.net/UpdateInfo.txt". Die Datei enthält zwei Textzeilen; in der ersten Zeile steht die Versionsnummer der aktuellen D-Fend Reloaded Version (z.B. "0.6.0" ) und in der zweiten Zeile die Download-Adresse für den Updateinstaller, der eine existierende D-Fend Reloaded Installation auf diese Version aktualisiert (z.B. "http://downloads.sourceforge.net/dfendreloaded/D-Fend-Reloaded-0.6.0-UpdateSetup.exe"). Wenn die Versionsnummer in der UpdateInfo.txt-Datei größer ist, als die Versionsnummer der momentan verwendeten D-Fend Reloaded Installation, wird Sie der Update-Checker fragen, ob Sie jetzt auf diese Version aktualisiern wollen. Wenn Sie diese Frage mit "Ja" beantworten, wird er die in der zweiten Zeile der in der UpdateInfo.txt-Datei angegebenen Datei in Ihr Verzeichnis für tempräre Dateien herunterladen und starten. Als letzten Schritt wird er Windows mitteilen, den Update-Installer so bald wie möglich wieder zu löschen. Da der Update-Installer jedoch gerade ausgeführt wird, kann Windows die Datei nicht sofort löschen und fügt sie in die interne "Beim nächsten Systemstart löschen"-Liste ein.

Momentan handelt es sich bei der UpdateInfo.txt-Datei um eine statische Datei, aber in Zukunft wäre es denkbar, dass der Webserver verschiedene Dateiinhalte abhängig von der momentan verwendeten D-Fend Reloaded Version ausliefert (ein kleiner Update-Installer für Aktualisierungen von Version 0.6.0 auf 0.6.1 und ein gröeres Installationspaket, welches alle älteren Versionen aktualisieren kann). Um dies zu ermöglichen, fügt D-Fend Reloaded beim Abfragen der Update-Informationsdatei ein "?Version=0.6.0" an die Adresse an (bzw. was sonst die aktuelle Verionsnummer ist). Wenn Sie nicht möchten, dass php-Skript auf dem Sourceforge-Webserver erfährt, welche D-Fend Reloaded Version Sie momentan einsetzen, können Sie diese Adressergänzung auf dieser Dialogseite deaktivieren. Da php-Skripte auf dem Sourceforge Webserver grundsätzlich keine Daten in Dateien abspeichern können (d.h. keine Anfragen in Log-Dateien erfassen können), ergeben sich keinerlei Einschränkungen der Privatephäre durch die Update-Abfrage.

Der Update-Checker stellt also lediglich eine Automatisierung dessen dar, was Sie manuell unternehmen würden, wenn Sie auf der Homepage nach Updates suchen. Die Update-Überprüfung kann automatisch beim Programmstart von D-Fend Reloaded durchgeführt werden. Da Programme, die ungefragt Internetverbindungen aufbauen häufig jedoch nicht gern gesehen werden, ist diese Option im Auslieferungszustand deaktiviert.

Die Update-Prüfung verwendet die Windows-Verbindungseinstellung (die Sie über die Programmeinstellungen des Internet Explorers anpassen können). Dies bedeutet insbesondere, dass D-Fend Reloaded den möglicherweise im Internet Explorer konfigurierten Proxy-Server verwenden wird.
