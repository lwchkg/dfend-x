Installer-Pakete für mehrere Spiele erstellen

D-Fend Reloaded kann Installationspakete von den bestehenden Spielen erstellen. Diese Pakete entalten alle Spieledateien (Programmdateien, Profildatei, Datanordner, aufgezeichnete Screensshots usw.) eignen sich daher gut zum Archivien und Übertragen von Spielen.

D-Femd Reloaded verwendet bei der Erstellung von Installer-Paketen das Opensource Installations-Ersteller System Nullsoft scriptable install system (NSIS). D-Fend Reloaded selbst erstellt lediglich eine nsi-Skriptdatei, aus der dann NSIS die eigentliche exe-Datei erstellt. Dies bedeutet, dass Sie NSIS auf Ihrem System installiert haben müssen, um diese Funktion nutzen zu können. (Die NSIS-Installation muss in D-Fend Reloaded nicht explizit eingetragen werden, da D-Fend Reloaded den NSIS-Installationspfad aus der Registry auslesen kann.)

Vollständige Spiele

Aus dieser Liste können Sie die Spiele, die in den Installer aufgenommen werden sollen, auswählen. Wenn Sie lediglich ein Installationspaket für ein einzelnes Spiel erstellen wollen, so können Sie auch die Funktion "Installer-Paket erstellen" aus dem "Profil"-Menü verwenden.

Vorlagen / Autosetup-Vorlagen

Sie können auch Installationspakete erstellen, die Vorlagen statt Spielen enthalten. Auf diesen beiden Dialogseiten können Sie die Vorlagen bzw. die Autosetup-Vorlagen wählen, die in das Paket aufgenommen werden sollen. (Ein Paket kann jeweils nur entweder Spiele, Vorlagen oder Autosetup-Vorlagen enthalten.)

Hinweis:

Da jeweils nut die Spiele bzw. Vorlagen von der jeweils sichtbaren Dialogseite in den Installer aufgenommen werden wenn Sie auf "Ok" klicken, müssen Sie die Elemente auf den jeweils nicht sichbaren Dialogseiten nicht explizit abwählen, damit sie nicht in den Installer aufgenommen werden.

Spiele im Installer nach Genres gruppieren

Wenn diese Option aktiviert ist, werden die Spiele bzw. Vorlagen nach Genres gruppiert in den Installer aufgenommen. Dies bedeutet, dass auf der Dialogseite "Spiele zur Installation auswählen" im Installer-Paket nicht nur eine einfache Auswahlbox angezeigt wird, sondern eine Baumstruktur, aus der die zu installierenden Spiele nach Genres sortiert ausgewählt werden können.

Paket-Einstellungen

Aus diesem Feld können Sie auswählen, ob D-Fend Reloaded lediglich ein nsi-Skript (für die spätere Verwendung mit NSIS) erstellten soll oder ob das erstellte nsi-Skript sofort mit NSIS zu einem Installer compiliert werden soll. (In den meisten Fällen dürfte die zweite Option die von Ihnen bevorzugte sein.)

Dateiname des Pakets

In diesem Eingabefeld können Sie den Namen des anzulegenden exe-Installers angeben. Die Datei muss in dem D-Fend Reloaded Basisverzeichnis (siehe Dialogseite Verzeichnisse des Programmeinstellungen-Dialogs) liegen, da in der Datei relative Pfadnamen verwendet werden, die nur dann gültig sind, wenn das nsi-Skript in diesem Verzeichnis liegt. (Der exe-Installer selbst hingegen kann an jede beliebige Position kopiert werden und auch von jedem Verzeichnis aus gestartet werden.)
