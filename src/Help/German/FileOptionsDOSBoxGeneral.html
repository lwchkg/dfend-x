Programmeinstellungen-Dialogseite "Globale DOSBox-Einstellungen"

Diese Dialogseite ist nur sichtbar, wenn Sie den Konfigurationsmodus "Vollständig" unten im Programmeinstellungen-Dialog gewählt haben.

Auf dieser Seite können Sie einstellen, die für alle eingerichteten DOSBox-Installationen gültig sind.

  • D-Fend Reloaded minimieren, wenn DOSBox gestartet wird
    Wenn diese Option aktiviert ist, wird das D-Fend Reloaded Hauptfenster minimiert, sobald ein DOSBox-basierendes Profil gestartet wird.
    (Voragebwert für diese Option: aus)
  • Programmfender wiederherstellen, wenn DOSBox beendet wird
    Diese Funktion kann nur aktiviert werden, wenn die Option "D-Fend Reloaded minimieren, wenn DOSBox gestartet wird" ebenfalls aktiviert ist. Diese Option stellt das D-Fend Reloaded Hauptfenster wieder her, sobald die DOSBox-Instanz, durch deren Start das Fenster minimiert wurde, wieder geschlossen wurde.
    (Voragebwert für diese Option: aus)
  • Kurze Pfadnamen in Mount-Befehlen verwenden
    DOSBox-Kommandozeilen dürfen maximal 134 Zeichen enthalten. Dies hat zur Folge, dass lange Mount-Befehle zum Einbinden von Laufwerken in DOSbox möglicherweise scheitern, wenn der Pfad, der als Laufwerk eingebunden werden soll, zu lang ist. Wenn diese Option aktiviert ist, wandelt D-Fend Reloaded beim Erzeugen der DOSBox-conf-Datei lange Pfadnamen in ihre kurzen Varianten um (z.B. würde "Program files" in "PROGRA~1" umgewandelt werden).
    (Voragebwert für diese Option: ein)
  • Immer conf-Dateien für alle Profile anlegen
    Ist diese Option aktiviert, so legt D-Fend Reloaded zu jeder Profil-Datei (*.prof) im "Confs"-Verzeichnis jeweils eine DOSBox-conf-Datei (*.conf) an und hält diese stets aktuell, so dass externe Programme diese verwenden können. Ist diese Option deaktiviert, so wird nur beim Aufruf von DOSBox dynamisch eine conf-Datei angelegt. (Diese Funktion ist nicht in der portablen Betriebsart verfügbar.)
    (Voragebwert für diese Option: aus)
  • Dialog zur Fehlerbehebung anzeigen, wenn DOSBox in weniger Sek. schließt als
    Über dieses Feld können Sie einstellen, wann der Dialog, über den DOSBox-Startprobleme behoben werden können, angezeigt werden soll. Wenn sich DOSBox innerhalb weniger Sekunden nach dem Start wieder beendet, kann davon ausgegangen werden, dass es ein Problem gab. Entweder konnte DOSBox selbst nicht korrekt gestartet werden oder aber das DOS-Programm innerhalb von DOSBox hat sich sofort wieder beendet. In beiden Fällen kann der DOSBox-Startprobleme-Dialog helfen, das Problem zu beheben.
    (Voragebwert für diese Option: 3)
