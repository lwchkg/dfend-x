Profileditor-Dialogseite "Profil"

Auf dieser Dialogseite können Sie die Basiseinstellungen für das Profil vornehmen. Der Inhalt dieser Seite variiert in Abhängigkeit von dem Typ des Profils, dass die gerade bearbeiten (DOSBox Profil, ScummVM Profil oder Profil für ein Windows-Spiel).

  • Icon auswählen
    Sie können für jedes Profil ein Icon festlegen. Das Icon wird in den meisten der verfügbaren Ansichtsmodi in der Spieleliste angezeigt. Icons können global über den Icons-Manager verwaltet werden. Allerdings können Sie auch eine individuelle beliebige Icondatei für das Profil festlegen. Falls Sie eine individuelle Icondatei verwenden, so können Sie für den Pfad zu dem Icon auch relative Pfadangaben verwenden, genauso wie auch für alle anderen Spieledateien.
  • Icon löschen
    Über diese Schaltfläche können Sie das gewählte Icon wieder aus dem Profil entfernen. Die Icondatei selber wird dabei nicht gelöscht (unabhängig davon, ob es sich um ein Icon aus der Icon-Manager Liste oder um ein individuell festgelegtes Icon handelt), sondern es wird lediglich der Eintrag in dem Profil entfernt.
  • Profilname
    Der Name, den Sie in diesem Eingabefeld festlegen können, wird in der Spieleliste zur Identifikation des Profils angezeigt. Im Gegensatz zu D-Fend v2 wird der hier eingegebene Name nicht gleichzeitig als Dateiname für das Profil verwendet, d.h. sie können beliebige Zeichen (einschließlich derer, die in Dateinamen nicht zulässig sind) in dem Profilnamen verwenden.
  • Dateiname
    In diesem Eingabefeld können Sie den Dateinamen (ohne Erweiterung), unter dem das Profil gespeichert werden soll angeben. Alle nicht zulässigen Zeichen werden beim Schließen des Dialogs automatisch aus dem Dateinamen entfernt. Beim Anlegen eines neuen Profils ist dieses Feld deaktiviert; der initiale Dateiname eines Profils wird stets automatisch aus dem Profilnamen abgeleitet.
  • Programmdatei (nur verfügbar für DOSBox und Windows-Profile)
    In dieser Box können Sie die zu startende Programmdatei und ihre Parameter festlegen. Der Dateiname wird, sofern möglich, relativ zu dem Basisverzeichnis (welches auf der Verzeichnisse Dialogseite des Programmeinstellungen-Dialogs festgelegt werden kann) gespeichert. D-Fend Reloaded übersetzt den Pfad in der System-Verzeichnisstruktur in einen Pfad in der DOSBox-Verzeichnisstruktur gemäß der eingebundenen DOSBox-Laufwerke beim Start von DOSBox. Wenn es sich beim dem Profil um ein DOSBox-Profil handelt, können Sie außerdem die Option Dateipfad relativ zur DOSBox-Verzeichnisstruktur interpretieren aktivieren. In diesem Fall wird D-Fend Reloaded den hier eingegebenen Pfad und Dateinamen als Pfad innerhalb des DOSBox-Verzeichnisstruktur interpretieren und wird nicht versuchen, ihn beim Aufruf von DOSBox aus der Windows-Verzeichnisstruktur in die DOSBox-Verzeichnisstruktur zu übersetzen. (Dies ist insbesondere hilfreich beim Arbeiten mit Images, da der Inhalt einer Image-Datei überhaupt nicht in der Windows-Verzeichnisstruktur zur Verfügung steht.)
  • Setupdatei (nur verfügbar für DOSBox und Windows-Profile)
    In dieser Box können Sie die zu startende Setupdatei und ihre Parameter festlegen. Der Dateiname wird, sofern möglich, relativ zu dem Basisverzeichnis (welches auf der Verzeichnisse Dialogseite des Programmeinstellungen-Dialogs festgelegt werden kann) gespeichert. D-Fend Reloaded übersetzt den Pfad in der System-Verzeichnisstruktur in einen Pfad in der DOSBox-Verzeichnisstruktur gemäß der eingebundenen DOSBox-Laufwerke beim Start von DOSBox. Wenn es sich beim dem Profil um ein DOSBox-Profil handelt, können Sie außerdem die Option Dateipfad relativ zur DOSBox-Verzeichnisstruktur interpretieren aktivieren. In diesem Fall wird D-Fend Reloaded den hier eingegebenen Pfad und Dateinamen als Pfad innerhalb des DOSBox-Verzeichnisstruktur interpretieren und wird nicht versuchen, ihn beim Aufruf von DOSBox aus der Windows-Verzeichnisstruktur in die DOSBox-Verzeichnisstruktur zu übersetzen. (Dies ist insbesondere hilfreich beim Arbeiten mit Images, da der Inhalt einer Image-Datei überhaupt nicht in der Windows-Verzeichnisstruktur zur Verfügung steht.)
  • Zusätzliche Programmdateien (nur verfügbar für DOSBox und Windows-Profile)
    Zusätzlich zu der Programm- und der Setupdatei können Sie in diesem Dialog weitere Programmdateien festlegen, die zu dem gewählten Spiel gehören (z.B. Level-Editoren usw.). Diese zusätzlichen Programmdateien können dann direkt über das Kontextmenü des Profils gestartet werden.
  • Keine Windows-Dateiwarnungen für dieses Profil (nur verfügbar für DOSBox-Profile)
    Normalerweise warnt Sie D-Fend Reloaded, wenn Sie versuchen, eine Windows-EXE-Datei mit DOSBox auszuführen, da DOSBox nicht in der Lage ist, Windows-Dateien auszuführen. Es gibt jedoch einige wenige Spiele, die kombinierte DOS/Windows-EXE-Dateien besitzen, die sowohl unter DOS als auch unter Windows ausgeführt werden können. D-Fend Reloaded würde den Windows-Anteil in diesen Dateien erkennen und folglich vor jedem Aufruf dieses Profils entsprechend warnen. Um diese in diesem Fall unnötigen Warnungen zu deaktivieren, kann diese Option aktiviert werden. (Dieser Schalter wird nur sichtbar, wenn es Programm- oder Setup-Datei eine Windows-EXE-Datei gewählt wurde.)
  • Spiel (nur verfügbar für ScummVM-Profile)
    In dieser Auswahlbox können Sie den Namen des zu startenden Spiels festlegen. (ScummVM benötigt diese Information, da es beim Starten der unterschiedlichen unterstützten Spiele unterschiedliche Einstellungen aktiviert.)
  • Verzeichnis des Spiels (nur verfügbar für ScummVM-Profile)
    In diesem Feld können Sie das Verzeichnis, in dem ScummVM nach den Dateien des Spiels suchen soll, festlegen.
  • Archivdatei in das Spiel-Verzeichnis entpacken (und später wieder einpacken) (nur verfügbar für ScummVM-Profile)
    Wenn diese Option aktiviert ist (und Sie in dem folgenden Eingabefeld eine Zip-Datei angegeben haben), wird D-Fend Reloaded diese Zip-Datei vor dem Aufruf von ScummVM in das angegebene Spiel-Verzeichnis entpacken. Nach dem Programmende von ScummVM wird der Inhalt des Verzeichnisses wieder inde die Zip-Datei eingepackt und das Verzeichnis wird gelöscht. Auf diese Art können Sie praktisch direkt Spiele in Zip-Dateien in ScummVM ausführen.
