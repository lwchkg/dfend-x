Hinzufügen mit Assistent

Mit Hilfe des Assistent des Assistenten können Sie auf sehr einfache Weise neue Profile in D-Fend Reloaded erstellen. Der Assistenten-Dialog besteht aus fünf Seiten, von denen je nach dem, welche Informationen D-Fend Reloaded über das Spiel bereits vorliegen, zwei bis fünf angezeigt werden:

Seite 1: Typ des neuen Profils

Auf der ersten Seite müssen Sie lediglich festlegen, ob es sich bei dem neu anzulegenden Profil um ein DOSBox-basierendes Profil, um ein ScummVM-basierendes Profil oder aber um ein Profil für ein Windows-Spiel handeln soll. (Die Option "Scumm-basierendes Adventure" wird nur angeboten, wenn Sie im Programmeinstellungen-Dialog den Pfad zu Ihrer ScummVM-Installation angegeben haben.) Nahezu alle DOS-Spiele können in DOSBox ausgeführt werden; ScummVM kann hingegen nur für Scumm-Adventures verwendet werden.

Zusätzlich können Sie einstellen, wie viele Fragen der Assistent Ihnen stellen soll. In der Standardeinstellung "Spiel automatisch konfigurieren, wenn eine passende AutoSetup-Vorlage existiert", wird der Assistent bereits nach den ersten zwei der fünf Seiten geschlossen, wenn das neue Spiel durch eine AutoSetup-Vorlage eindeutig identifiziert wurde und keine weiteren Benutzereinstellungen notwendig sind. Alternativ kann eingestellt werden, dass entweder immer alle fünf Seiten des Assistenten angezeigt werden sollen oder aber, dass der Assistent auch dann bereits nach der zweiten Seite geschlossen wird, wenn keine Informationen über das Spiel vorliegen. (In diesem Fall wird D-Fend Reloaded versuchen, das Spiel gemäß der Standardvorlage zu konfigurieren.)

Seite 2: Programmdateien (für neue DOSBox-Profile)

Auf dieser Seite können Sie die Programmdatei und optional eine Setup-Datei für das Profil festlegen. Die auf dieser Dialogseite festgelegte Programmdatei wird in DOSBox gestartet, wenn Sie später auf den Profileintrag in der Spieleliste doppelt klicken.

Damit D-Fend Reloaded mit relativen Pfaden arbeiten kann, sollten Sie die Spieledateien in einem Unterordner des "Empfohlenen Übergeordneten Verzeichnisses ablegen. (Siehe auch Einführung in D-Fend Reloaded für eine Beschreibung der empfohlenen Verzeichnisstruktur.) Der auf dieser Dialogseite angebotenen Spieleordner kann auf der Verzeichnisse-Seite des Programmeinstellungen-Dialogs angepasst werden.

Wenn Sie auf Zusätzliche Dateien manuell konfigurieren klicken, wird der Dialog um ein Eingabefeld erweitert, übder dass Sie einen Ordner für zusätzliche Spieledaten wie Handbücher usw. konfigurieren können. Der Inhalt dieses Ordners wird später unten im D-Fend Reloaded Hauptfenster angezeigt, wenn Sie das Profil in der Spieleliste anklicken. Alternativ können Sie auch die Option Zusätzliche Datenordner automatisch festlegen aktivieren. In diesem Fall legt D-Fend Reloaded automatisch basierend auf dem Namen des Profils einen Datenordner an.

Seite 2: Programmdateien (für neue ScummVM-Profile)

Auf dieser Seite können Sie den Ordner festlegen, in dem sich das auszuführende Scumm-Spiel befindet. Der hier angegebene Ordnername wird ScummVM beim Aufruf übergeben.

Damit D-Fend Reloaded mit relativen Pfaden arbeiten kann, sollten Sie die Spieledateien in einem Unterordner des "Empfohlenen Übergeordneten Verzeichnisses ablegen. (Siehe auch Einführung in D-Fend Reloaded für eine Beschreibung der empfohlenen Verzeichnisstruktur.) Der auf dieser Dialogseite angebotenen Spieleordner kann auf der Verzeichnisse-Seite des Programmeinstellungen-Dialogs angepasst werden.

In der mit Spiel beschrifteten Auswahlbox können Sie einstellen, für welches Spiel Sie ein Profil erstellen wollen. Die Information, um welches Spiel es sich handelt wird beim Aufruf ebenfalls an ScummVM übergeben.

Wenn Sie auf Zusätzliche Dateien konfigurieren klicken, wird der Dialog um ein Eingabefeld erweitert, übder dass Sie einen Ordner für zusätzliche Spieledaten wie Handbücher usw. konfigurieren können. Der Inhalt dieses Ordners wird später unten im D-Fend Reloaded Hauptfenster angezeigt, wenn Sie das Profil in der Spieleliste anklicken.

Seite 2: Programmdateien (für neue Windows-Spiele Profile)

Auf dieser Seite können Sie die Programmdatei und optional eine Setup-Datei für das Profil festlegen. Die auf dieser Dialogseite festgelegte Programmdatei wird gestartet, wenn Sie später auf den Profileintrag in der Spieleliste doppelt klicken.

Wenn Sie auf Zusätzliche Dateien konfigurieren klicken, wird der Dialog um ein Eingabefeld erweitert, übder dass Sie einen Ordner für zusätzliche Spieledaten wie Handbücher usw. konfigurieren können. Der Inhalt dieses Ordners wird später unten im D-Fend Reloaded Hauptfenster angezeigt, wenn Sie das Profil in der Spieleliste anklicken.

Seite 3: Einstellungen (für neue DOSBox-Profile)

Auf dieser Seite können Sie die wichtigsten Einstellungen für den Start des Spiels in DOSBox vornehmen. Ein Spiel mit Hilfe des Assistenten zu konfigurieren besteht im Wesentlichen daraus, eine passende Vorlage für das neue Profil zu wählen. Es gibt insgesamt 5 verschiedene Arten, wie eine Vorlage gewählt werden kann:

  • Passende Autosetup Vorlage verwenden (empfohlen)
    Die Vorlagen in dieser Liste sind kompatibel mit dem gewählten Spiel. die Kompabilität wurde mit Hilfe einer Prüfsumme festgestellt. Wenn es eine Vorlage in dieser Auswahlbox gibt, so wird daher empfohlen, diese zu verwenden.
  • Autosetup Vorlage, welche nur vom Dateinamen her passt, verwenden
    Die Vorlagen in dieser Liste sind passen vom Dateinamen her zu dem gewählten Spiel, nicht jedoch von der Prüfsumme her. D.h. dass Sie möglicherweise eine andere Version des Spieles verwenden, als die, für die die Vorlage erstellt wurde. Wenn in der ersten Auswahlbox keine kompatiblen Vorlagen angezeigt werden, aber in dieser Auswahlbox eine Vorlage vorhanden ist, die auch vom Namen des Spiels her zu passen scheint, so sollten Sie diese Vorlage verwenden.
  • Beliebige Autosetup Vorlage verwenden
    In dieser Auswahlbox werden unabhängig von der gewählen Programmdatei stets alle Autosetup Vorlagen angezeigt. Wenn in den obigen beiden Auswahlboxen keine Einträge vorhanden sind, aber eine der Vorlagen aus dieser Auswahlbox den Namen des Spiels, welches Sie gerade hinzufügen wollen trägt, so können Sie diese Vorlage verwenden. Wenn Sie eine dieser Vorlagen jedoch auf ein Spiel anwenden, für welches sie nicht erstellt wurde, so kann dies dazu führen, dass sich das Spiel nicht starten lässt. Außerdem stimmen in diesem Fall die zusätzlichen Spieleinformationen wie Genre, Entwickler usw. nicht.
  • Benutzervorlage verwenden
    In dieser Auswahlbox werden unabhängig von der gewählen Programmdatei stets alle benutzerdefinierten Vorlagen angezeigt. Wenn es keine zu Ihrem Spiel passende Autosetup Vorlage gibt, so können Sie eine dieser Vorlagen wählen. Diese Vorlagen enthalten keine spielespezifischen Einstellungen und es sollte damit möglich sein, die meisten Spiele auszuführen.
  • Standard-Assistentenvorlage verwenden
    Wenn sie diese Option wählen, wird eine fest in den Assistenten eingebaute Basis-Vorlage verwendet, die ebenfalls für die meisten Spiele verwendbar sein sollte und ebenfalls keine spielespezifischen Einstellungen aufweist. Über die Auswahlfelder unterhalb dieser Option können Sie einige grundlegende Einstellungen an dieser Vorlage vornehmen.

Seite 3: Einstellungen (für neue ScummVM-Profile)

Auf dieser Dialogseite können sie einige zusätzliche Einstellungen für das neue Profil vornehmen. Welche Einstellungen hier konkret angeboten werden, hängt von dem auf der vorherigen Seite ausgewählten Spiel ab.

Seite 4: Spieleinformationen

Auf dieser Seite können Sie zusätzliche Informationen zu dem Spiel (wie Name, Genre, Entwickler, Herausgeber usw.) angeben. Diese Informationen werden dann in der Spieleliste (im Ansichtsmodus Details) angezeigt und die Profile können nach diesen Informationen gefiltert werden.

Über die "Spiel nachschlagen bei ..." Schaltfläche können Sie auf nach dem Namen des Profil auf verschiedenen, teilweise auf alte Spiele spezialisierten Suchmaschinen suchen.

Wenn Sie ein DOSBox-Profil hinzufügen und eine Autosetup Vorlage gewählt haben, so wird diese Dialogseite nicht angezeigt, da die Autosetup Vorlagen bereits alle relevanten Spieleinformationen enthalten.

Seite 5: Virtuelle Laufwerke (für neue DOSBox-Profile)

Auf dieser Dialogseite werden die virtuellen Laufwerke, die DOSBox verwenden soll, angezeigt. Die Spiele-Programmdatei muss über eines dieser Laufwerke in DOSBox erreichbar sein, um gestartet werden zu können. D-Fend Reloaded versucht basierend auf dem Pfad des Spieles und ein Einstellungen in der Vorlage automatisch zu bestimmen, welche virtuellen Laufwerke anzulegen sind. Wenn die auf dieser Seite angezeigten Laufwerke jedoch für den Start des Spiels nicht ausreichend sind (z.B. weil zusätzlich ein CD-Laufwerk verwendet werden soll), so können Sie hier noch Ändernungen an der Laufwerkskonfiguration vornehmen. (Selbstverständlich können Sie die Konfiguration auch jederzeit später über den Profileditor-Dialog anpassen.)
